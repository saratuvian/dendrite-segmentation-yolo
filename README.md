# Dendrite Segmentation in SEM Images

## Project Overview

This project investigates automatic segmentation of dendritic structures in SEM (Scanning Electron Microscopy) images.

Two approaches are implemented:

1. Classical image processing pipeline  
2. Deep learning model using YOLO segmentation  

The goal is to compare traditional methods with deep learning for detecting dendrites in noisy SEM images.

---

## Dataset

- 14 SEM images  
- Annotated using Roboflow (polygon segmentation)  
- Exported in YOLO segmentation format  

Two groups:

**EASY**
- clean background  
- clear dendrites  

**HARD**
- noisy SEM images  
- complex structures  

Roboflow project:  
https://app.roboflow.com/saras-workspace-fd6vg/sem_dendrite_segmentation_v2/3

## Notebooks Directory Structure

All notebooks and required resources are located inside the `notebooks/` directory due to the use of relative paths.

The structure inside `notebooks/` is as follows:

- **data/**  
  Contains the original SEM images before preprocessing (no cropping applied).

- **dataset/**  
  Contains the dataset exported directly from Roboflow in YOLO segmentation format (train/valid/test splits and labels).

- **datasets/**  
  Contains the processed dataset used in the project (after preprocessing such as bottom cropping + normalization).

- **outputs/**  
  Contains results generated by the notebooks, including:
  - binary masks  
  - overlay visualizations  
  - skeleton images  
  - evaluation CSV files  

- **YOLO model files (.pt)**  
  Intermediate or local model files used during experimentation (not required for final reproduction).

> Note:  
> This structure is required because the notebooks rely on relative paths.  
> Running the notebooks from outside this directory may require adjusting file paths.

## Classical Pipeline

Main steps:

1. Normalization  
2. Bottom crop (93 px)  
3. Denoising  
4. Background flattening  
5. Sauvola threshold  
6. Morphological cleanup  
7. Watershed separation  
8. Skeletonization  

Outputs:
- binary masks  
- overlays  
- skeletons

## YOLO Model

### Baseline
YOLOv11n-seg  
- epochs: 50  
- image size: 640  
- batch size: 4  

### Final Model
YOLOv11s-seg  
- epochs: 150  
- image size: 640  
- batch size: 4

## Evaluation

Metrics:
- IoU  
- Dice  
- Precision  
- Recall  

Results:

| Method | IoU | Dice | Precision | Recall |
|------|------|------|------|------|
| Classical | 0.545 | 0.695 | 0.733 | 0.702 |
| YOLO | 0.584 | 0.729 | 0.716 | 0.763 |

Conclusion:  
YOLO improves IoU, Dice and Recall, while the classical pipeline has slightly higher Precision.
